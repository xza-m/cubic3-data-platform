# 语义工作台建模页视觉收敛设计

> 日期：2026-04-07
> 主题：在不改变语义工作台主结构的前提下，将建模页视觉与布局密度收敛到统一的平台工作台风格

## 1. 背景

当前语义工作台已经完成了主结构收口：

- 左侧是资源库
- 顶部有流程提示、上下文信息和对象标题
- 中间保留 `可视化 / YAML / Python`
- 下方按 `维度 / 指标 / 日期属性` 展示字段分组

但当前建模页仍存在明显的视觉问题：

1. 顶部同时出现流程条、三张上下文卡、对象标题和 AI 摘要卡，页面层级过散
2. 字段区以大卡片堆叠为主，纵向占用过高，缺少工作台的紧凑感
3. 左侧资源区和右侧建模区像两套不同产品，风格不统一
4. 大量边框、圆角和松散留白让页面更像“展示页”，而不是“编辑工作台”

用户已明确要求：

- 只改视觉与布局密度
- 不改变语义工作台的主结构
- 整体风格要回到平台统一的工作台语言
- 气质参考：`查询中心的工作台感 + 数据中心的克制密度`

## 2. 设计前提

本设计的边界非常明确：

- 不改左侧资源库与右侧建模工作区的主结构
- 不改 `可视化 / YAML / Python / 预览 / 发布` 的职责
- 不引入新的建模流程或交互模型
- 只调整视觉层级、布局密度、卡片样式、信息组织方式

工程原则如下：

- `KISS`：只解决视觉与信息密度问题，不借机重构流程
- `YAGNI`：不增加新的面板、向导或复杂交互
- `SOLID`：强化页面层级，让状态、对象、字段三层语义更清楚
- `DRY`：继续复用现有区块，不再额外引入新的说明面板或平行信息区

## 3. 目标与非目标

### 3.1 目标

- 让语义建模页从“松散展示页”收紧为“平台统一的编辑工作台”
- 降低顶部视觉噪音，让对象标题和核心操作更清楚
- 提高字段列表扫读效率
- 统一左侧资源区和右侧建模区的视觉语言

### 3.2 非目标

- 不重画信息架构
- 不重做 AI 建模链路
- 不新增新的建模模式
- 不修改后端接口或字段语义

## 4. 方案比较

### 方案 A：轻样式修补

做法：

- 保留当前所有区块和卡片
- 只做颜色、边框和间距微调

优点：

- 成本最低

缺点：

- 无法解决“卡片太多、层级太碎”的根因
- 页面仍然缺少工作台气质

### 方案 B：工作台式视觉收敛

做法：

- 保留现有主结构
- 重新组织顶部层级
- 将 AI 摘要区降级为轻提示 + 紧凑统计
- 将字段区从“详情卡堆叠”改为“按组管理的紧凑列表块”
- 统一左侧资源区与右侧建模区的视觉语言

优点：

- 最符合平台统一目标
- 风险可控
- 不需要改变现有交互骨架

缺点：

- 需要较系统地重做样式与排版，而不是局部补丁

### 方案 C：强编辑器风格

做法：

- 让建模页更像 IDE 或代码编辑器
- 使用更强的分区、标题条和对比度

优点：

- 工作台感最强

缺点：

- 容易与查询中心过于相似
- 会偏离当前平台整体的克制风格

### 推荐结论

采用 **方案 B：工作台式视觉收敛**。

## 5. 视觉改版设计

### 5.1 顶部区域收紧

顶部从“提示卡 + 指标卡 + 标题卡”的组合，收敛为三层：

1. 轻量流程辅助条
2. 紧凑上下文信息带
3. 标准页头

具体规则：

- 流程条继续保留，但视觉降级为辅助导航，不再像主操作胶囊
- `当前资源 / 当前 Cube / 当前状态` 不再使用三张大卡片，而改为一条紧凑信息带
- 页面标题与来源合并为标准对象页头，右侧保留 `预览 / 发布`
- 顶部减少独立卡框数量，主要用浅色背景和分隔线建立层级

### 5.2 AI 摘要区收紧

AI 摘要区从主视觉大卡改为：

1. 一条轻量状态提示：`AI 已生成建模初稿`
2. 一组紧凑摘要统计：字段总数、维度/日期、指标
3. 一句短操作提示：`先确认字段分类与描述，再进入预览验证`

不再保留：

- 过长说明文案
- 占位过大的蓝色主卡
- 像首页 KPI 卡一样的强视觉统计块

### 5.3 字段分组区改为紧凑列表块

保留按组展示：

- 维度
- 指标
- 日期属性

但表现方式改为：

- 分组块而不是厚重白卡
- 组内以紧凑字段项列表呈现
- 组标题 + 数量 + 轻操作（如 `添加`）统一排布

字段项的默认信息缩减为两层：

- 第一行：字段名、分类标签、识别结果、字段类型
- 第二行：描述摘要

以下信息不再默认大面积展开：

- 来源类型
- 识别置信度
- 推荐理由

这些内容应降为折叠详情或弱信息区。

### 5.4 左右统一

左侧资源库和右侧建模区统一为同一套工作台视觉语言：

- 左侧更像导航面板，不再是松散资产卡片
- 右侧更像主编辑区，不再是层层卡片堆叠
- 两侧共享统一的：
  - 圆角体系
  - 边框强度
  - 留白节奏
  - 标签语义

同时要求：

- 左侧选中资源态与右侧页头中的来源信息形成视觉呼应
- 左右两侧的内容密度相互匹配，避免“一边很密、一边很松”

## 6. 视觉规范清单

### 6.1 面板与卡片

- 减少大卡片数量
- 面板级使用中等圆角
- 行项级使用小圆角
- 弱化边框，优先用分区和留白表达层级

### 6.2 标题层级

- 一级：当前 Cube 标题
- 二级：来源、状态、当前资源
- 三级：字段分组标题
- 四级：字段项元信息

不允许统计数字、说明文案与对象标题抢同一级视觉。

### 6.3 按钮优先级

- 主按钮：`发布`
- 次按钮：`预览`
- 弱操作：`添加`、展开详情等

同屏不应出现过多高权重按钮。

### 6.4 标签语义

分类标签、状态标签、辅助标签必须使用不同视觉权重，避免全部做成同一种胶囊样式。

### 6.5 空白与留白

留白必须服务于：

- 分组
- 扫读
- 操作定位

不允许为了“通透感”留下大面积无功能空白。

## 7. 验收标准

### 7.1 页面感知

- 用户进入建模页后，能明确感知这是“工作台编辑页”而不是“展示卡片页”
- 顶部层级清楚，不再出现多个视觉中心并列竞争

### 7.2 扫读效率

- 字段很多时仍能快速浏览
- 默认视图可以快速判断字段分类是否正确
- 缺失描述字段不会伪装成已完成信息

### 7.3 风格统一

- 左侧资源区与右侧建模区风格一致
- 语义工作台与平台其他工作台页面语言一致

## 8. 实施顺序

1. 先改顶部区域与上下文带
2. 再改 AI 摘要区
3. 再改字段分组与字段项密度
4. 最后统一左侧资源区样式

## 9. 结论

本轮改版不是重画语义工作台，而是在既有结构上做一次 **工作台式视觉收敛**：

- 保留结构
- 收紧层级
- 降低卡片感
- 提高扫读效率
- 回归平台统一风格

最终目标是让语义建模页更像一个真实可用的编辑工作台，而不是一组松散的展示型卡片拼在一起。
