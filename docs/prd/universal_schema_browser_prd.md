📚 通用元数据浏览器 - 功能设计说明书
1. 产品定位
该组件是数据平台的导航罗盘。它提供了一个统一的、树状的视图，用于在异构数据源（MaxCompute、MySQL、PostgreSQL 等）之间进行层级化浏览、检索和操作。

2. 核心功能模块
2.1 数据源连接与切换
多源支持：支持挂载多种不同类型的数据库实例。

快速切换：提供入口（如下拉选单），允许用户在已注册的不同数据源实例之间快速切换上下文。

连接状态感知：在切换数据源时，应能直观反馈当前连接是否健康（如：连接成功、连接超时、认证失败）。

2.2 层级化树形导航 (Hierarchical Navigation)
统一逻辑层级：屏蔽底层数据库的物理差异，对外统一抽象为标准层级结构：

Level 1: 实例/数据源 (Instance)

Level 2: 库/项目 (Database / Project)

Level 3: 模式 (Schema) [注：仅针对支持 Schema 的数据库，如 PG]

Level 4: 实体对象 (Table / View / Materialized View)

Level 5: 属性明细 (Columns / Partitions)

按需加载 (Lazy Loading)：

初始状态仅加载顶层节点。

仅当用户展开某个节点（如点击“库”）时，才实时请求后端获取该节点下的子内容（如“表”列表）。

有效支撑单一库下包含数万张表的超大规模场景。

2.3 检索与过滤 (Search & Filter)
关键字搜索：支持输入表名、字段名或备注的关键字进行搜索。

混合搜索策略：

本地过滤：优先在当前已展开/已加载的节点中进行快速筛选。

全局检索：支持触发后端接口，在整个数据源范围内进行模糊匹配，并定位到具体路径。

对象类型过滤：支持仅查看“物理表”或仅查看“视图”。

2.4 交互操作 (User Interactions)
此组件需支持三种不同深度的交互模式，以适应不同的使用场景（如 SQL 编辑、新建数据集、数据探查）：

单击交互 (Selection)：

行为：选中某个节点。

结果：高亮当前行，并向外部容器发送信号（例如：通知右侧面板加载该表的详情、预览数据或 Schema 信息）。

双击交互 (Quick Insertion)：

场景：主要用于 SQL 编辑器模式。

行为：双击表名或字段名。

结果：自动将该对象的完整引用路径（如 project.table 或 column_name）插入到代码编辑器的光标位置。

拖拽交互 (Drag & Drop)：

场景：SQL 编写或可视化建模。

行为：按住节点拖动到外部区域。

结果：携带该节点的元数据（表名、字段类型等）并在目标区域生成对应的文本或图形实体。

2.5 上下文菜单 (Context Menu)
用户在节点上点击右键时，需提供快捷功能入口：

复制操作：复制表名、复制完整路径、复制字段名。

生成 SQL：

Generate SELECT：自动生成查询前 100 条数据的 SQL 模板。

Generate DDL：获取建表语句。

快速预览：弹窗展示该表的前 N 行数据（无需写 SQL）。

刷新：强制重新加载当前节点下的元数据（用于感知外部变更）。

2.6 元数据可视化标识
为了让用户一眼识别数据属性，需通过视觉符号（图标/标签）区分对象类型：

对象类型标识：区分 数据库、文件夹、表、视图。

字段类型标识：区分 字符串 (String)、数值 (Number)、时间 (Date/Time)、布尔 (Bool)。

关键属性标识：

🔑 主键 (Primary Key)

🧩 分区键 (Partition Key)

🔗 外键 (Foreign Key)

3. 状态管理与缓存
状态保持：当用户切换到其他页面再返回时，应尽量保持之前的展开层级和选中状态，避免用户重复点击。

主动刷新：提供顶级的“全局刷新”功能，用于清除本地元数据缓存，重新从数据库拉取最新结构。

总结
该功能模块的核心在于**“连接”与“发现”**。它不负责数据的计算，而是作为用户探索数据资产的第一入口，要求响应迅速、结构清晰、且能与 SQL 编辑器或配置表单无缝联动。
