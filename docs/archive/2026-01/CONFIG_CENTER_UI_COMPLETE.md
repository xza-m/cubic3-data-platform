## 🎯 使用场景

### 场景 1: 配置飞书推送
1. 创建飞书渠道
2. 填写 chat_id
3. 创建订阅，选择应用和事件类型
4. 启用订阅
5. 应用执行完成后自动推送到飞书

### 场景 2: Webhook 集成
1. 创建 Webhook 渠道
2. 配置 URL、Method、Headers
3. 创建订阅
4. 事件触发时 POST 数据到 Webhook URL

### 场景 3: 多渠道推送
1. 创建多个渠道（飞书、Webhook）
2. 为同一应用创建多个订阅
3. 一个事件同时推送到多个渠道
