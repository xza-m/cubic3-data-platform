# 历史归档目录

本目录保存迁移记录、阶段总结、问题修复和实施过程文档。
这些资料用于追溯背景和实现过程，不直接作为当前实现标准。

## 目录索引

- [2026-01 归档索引](2026-01/README.md)
- [历史专题目录](legacy/README.md)

## 适用范围

适合以下场景：

- 追溯某次重构、迁移或专项治理的背景
- 查找某个模块曾经的实施路径或问题定位过程
- 为新的设计或重构提供历史上下文

不适合以下场景：

- 判断当前系统如何启动
- 判断当前技术栈、路由、端口和命令
- 作为当前 API、页面或脚本的唯一依据

## 使用规则

- 看历史前，先看 `../DOC_ALIGNMENT_REPORT.md`
- 如果历史文档与当前代码冲突，以当前代码和基线文档为准
- 新的阶段性总结、迁移纪要和一次性修复记录，应继续归档到这里，而不是塞进首页
