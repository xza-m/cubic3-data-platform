开发应用与推送

应用中心把语义分析能力封装成可运营的实例，例如飞书问答、数据集卡片推送、异常监控和日报周报。这里关注实例创建、渠道连接、订阅触发和执行监控。

20 min Beginner 应用中心

应用生命周期

从应用定义到执行结果，所有页面的跳转和右侧上下文都应保持一致。

1

选择应用模板

在应用市场中选择模板。卡片应等高排列，类别、状态和实例数都能快速扫描。

2

创建应用实例

实例负责保存具体配置，例如数据集、推送目标、执行方式和负责人。点击实例时在当前上下文打开详情，不跳出模块壳层。

3

配置渠道和订阅

渠道承载飞书、钉钉、邮件或 Webhook 连接；订阅定义什么时候触发、触发什么实例、推到哪里。

{
  "method": "POST",
  "url": "https://open.feishu.cn/open-apis/bot/v2/hook/...",
  "retry": { "max_attempts": 3 }
}
4

监控执行记录

执行监控必须保留应用侧边栏，列表分页展示。点击记录时右侧展示摘要，详情按钮仍在应用模块内跳转。

上线前检查

应用不是只跑通一次，还要可追踪、可重试、可审计。

✓

配置可读

实例详情中能直接看到关键配置，不需要跳到多个页面拼上下文。

✓

失败可定位

最近状态、执行 ID、耗时和错误信息都能在执行记录中找到。

✓

入口一致

应用列表、应用实例、执行监控、渠道和订阅都使用一致的列表加上下文详情模式。
