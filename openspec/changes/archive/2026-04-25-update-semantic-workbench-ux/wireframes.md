## 页面级线框说明

本文档面向前端开发，描述 `CubeList`、`CubeStudio`、`DomainCanvas`、`DevTools` 四个核心页面的布局骨架、视觉层级、组件职责和交互重点。所有页面基于现有 `Semantic workbench` 组件体系推进，不重做全局导航和路由。

---

## 1. CubeList
### 页面目标
- 帮用户快速识别“哪些 Cube 需要处理”
- 支持高密度筛查、快速预览、跳转详情/编辑
- 避免做成传统 CRUD 或卡片浏览页

### 桌面布局
```text
+----------------------------------------------------------------------------------+
| SemanticPageHeader                                                               |
| 标题: Cube 管理   范围: 语义中心   状态摘要: 待处理 X / 草稿 X / 失败 X        |
| [新建 Cube] [批量筛选]                                                           |
+----------------------------------------------------------------------------------+
| SemanticStatusBanner (仅有阻塞或提醒时出现)                                      |
+----------------------------------------------------------------------------------+
| SemanticSurface                                                                  |
| +------------------------------------------------------------------------------+ |
| | SemanticToolbar                                                              | |
| | 搜索 | 状态 | 绑定状态 | 领域 | 负责人 | 标签 | 排序 | 快筛 chips           | |
| +------------------------------------------------------------------------------+ |
| +------------------------------------------------------+ +--------------------+ |
| | CubeTable                                            | | PreviewPanel       | |
| | 名称/技术名 | 状态 | 来源 | 领域 | 复用 | 更新时间 | | 当前状态            | |
| | 左侧状态轨                                          | | 来源绑定            | |
| | 行内主动作: 查看/编辑                               | | 领域/视图引用       | |
| | 次动作: 更多                                        | | 最近发布/变更       | |
| +------------------------------------------------------+ | 下一步动作         | |
|                                                          +--------------------+ |
+----------------------------------------------------------------------------------+
```

### 移动/窄屏适配
- 右侧 `PreviewPanel` 降级为抽屉，不隐藏核心动作
- 工具栏折叠为“搜索 + 筛选”两层
- 表格降级为紧凑列表，但保留状态轨、对象名、状态、主动作

### 视觉要求
- 主体是表格，不再用对象卡片堆叠
- 行左侧用 2px-3px 状态轨表达问题优先级
- 表格背景保持平整，选中行用轻微底色和边线高亮，不用整块彩色卡片
- 快筛 `chips` 比普通筛选更醒目，但低于主按钮

### 组件拆分
- `CubeToolbar.tsx`
- `CubeTable.tsx`
- `CubeTableRowStatus.tsx`
- `CubePreviewPanel.tsx`
- `useCubeListFilters.ts`
- `semantic-status.ts`

### 关键交互
- 默认排序为“需处理优先”
- 行点击更新右侧预览，不强制路由跳转
- 双击或主动作进入详情/编辑
- 快筛支持：`待处理`、`高复用`、`最近变更`

## 2. CubeStudio
### 页面目标
- 让新建/编辑 Cube 成为阶段式建模流程
- 一次只强调一个任务，避免长表单堆叠
- 把校验、预览、发布动作收敛到统一闭环

### 桌面布局
```text
+----------------------------------------------------------------------------------+
| SemanticPageHeader                                                               |
| 标题: Cube Studio / 当前 Cube 名称                                               |
| 状态: 草稿 / 待发布 / 已发布 / 校验失败                                          |
| [保存草稿] [发布]                                                                |
+----------------------------------------------------------------------------------+
| SemanticStatusBanner                                                             |
| 当前阻塞、校验提醒、与领域关系                                                   |
+----------------------------------------------------------------------------------+
| Grid: 280px / minmax(0,1fr) / 320px                                              |
| +----------------------+ +--------------------------------------+ +------------+ |
| | SemanticStepRail     | | CurrentStepPanel                     | | Inspector  | |
| | 1 基础信息           | | 步骤标题 + 说明                      | | 当前状态   | |
| | 2 来源绑定           | | 该步骤唯一主任务区                  | | 命名规范   | |
| | 3 维度/指标          | | Schema / 表单 / 结构预览            | | 引用关系   | |
| | 4 语义规则           | | 底部上一步/下一步                   | | 阻塞项     | |
| | 5 校验与预览         | |                                      | | 操作提示   | |
| | 6 保存与发布         | |                                      | |            | |
| +----------------------+ +--------------------------------------+ +------------+ |
+----------------------------------------------------------------------------------+
```

### 各步骤的面板样式
- `基础信息`：常规输入布局，信息密度低，留白更大
- `来源绑定`：`SchemaBrowser + 当前绑定摘要条`
- `维度/指标`：`自动识别结果 / 人工调整` 双 tab
- `语义规则`：基础规则默认展开，高级规则折叠
- `校验与预览`：`SemanticIssueList + 结构摘要 + 引用影响`
- `保存与发布`：动作确认与结果预览，不再混入编辑字段

### 视觉要求
- 当前步骤面板是视觉中心，步骤轨退后但清晰
- 不要在每个步骤里再套一层层卡片
- `Inset Surface` 用于 Schema、预览表、规则详情、IssueList
- 步骤切换动效只保留透明度和轻位移反馈

### 组件拆分
- `CubeStudioStepRail.tsx`
- `CubeStudioStepHeader.tsx`
- `CubeStudioBasicStep.tsx`
- `CubeStudioBindingStep.tsx`
- `CubeStudioStructureStep.tsx`
- `CubeStudioRulesStep.tsx`
- `CubeStudioValidationStep.tsx`
- `CubeStudioInspector.tsx`

### 关键交互
- 步骤切换保留未提交本地状态
- 任一步骤出现阻塞时，在步骤轨上显示红点或状态标记
- 发布前自动聚焦到 `校验与预览`
- 从 `领域建模` 回流时，页头展示来源上下文

## 3. DomainCanvas
### 页面目标
- 让领域建模页具备专业建模工作台体验
- 突出画布中心，降低左右侧栏视觉干扰
- 让 Join 编辑、冲突识别、发布检查形成闭环

### 桌面布局
```text
+--------------------------------------------------------------------------------------+
| SemanticPageHeader                                                                   |
| 当前领域 | 状态 | 最近发布 | [发布] [自动布局] [校验] [添加 Cube]                    |
+--------------------------------------------------------------------------------------+
| SemanticStatusBanner                                                                 |
| 发布前阻塞 / Join 缺失 / 草稿 Cube / 未绑定来源                                      |
+--------------------------------------------------------------------------------------+
| DomainWorkbenchContextBar                                                            |
| 实体数 | Join 数 | 异常数 | 当前筛选 | 当前选中对象                                  |
+--------------------------------------------------------------------------------------+
| Grid: 280px / minmax(0,1fr) / 340px                                                  |
| +----------------------+ +------------------------------------------+ +------------+ |
| | CubeLibrary          | | GraphCanvas                              | | Inspector  | |
| | 搜索                 | | 顶部浮层: 缩放/图例/筛选                | | Join 概览  | |
| | 全部/未入域/高复用   | | React Flow 画布                          | | 字段映射   | |
| | Cube 列表            | | 节点 / 连线 / MiniMap / Controls        | | 类型/基数  | |
| |                      | | 空画布引导                               | | 说明/保存  | |
| +----------------------+ +------------------------------------------+ +------------+ |
+--------------------------------------------------------------------------------------+
```

### 节点样式规范
- 节点为扁平化工作节点，不使用渐变、投影强调
- 信息分四层：
  - 显示名
  - 技术名
  - 来源绑定
  - 维度/指标/Join 数
- 选中态：边框加深 + 轻底色
- 冲突态：顶部或侧边风险条，不整块红底
- 高影响/高复用态：微弱标签，不抢占错误状态

### 连线样式规范
- 普通 Join：中性色实线
- 待补字段：虚线
- 冲突：橙红高亮
- 基数 `1:1 / 1:N / N:N` 用中段小标签表达
- 选中连线时，提高透明度和粗细，不改变全局色彩体系

### Inspector 结构
- `JoinInspector`
  - 关系概览
  - 字段映射
  - Join 类型 / 基数
  - 聚合策略
  - 说明
  - 保存 / 删除
- `CubeInspector`
  - Cube 摘要
  - 来源、字段数、引用关系
  - 跳转编辑
- `DomainInspector`
  - 当前领域状态
  - 实体规模
  - 阻塞项
  - 发布建议

### 视觉要求
- 画布区域是第一视觉层，左右栏使用更轻底色和更紧密排版
- 允许轻微网格背景，但不能变成炫技图谱
- 空态采用三步操作引导，不用大插画
- 图例、筛选、异常视图优先做成轻量浮层，不额外占大块布局

### 组件拆分
- `DomainWorkbenchContextBar.tsx`
- `DomainCubeLibrary.tsx`
- `DomainGraphCanvas.tsx`
- `DomainGraphLegend.tsx`
- `DomainJoinInspector.tsx`
- `DomainSummaryInspector.tsx`
- `useDomainCanvasFilters.ts`

### 关键交互
- 选中节点时高亮相邻连线
- 选中连线时右侧直接进入 Join 编辑
- 支持“仅看异常”“仅看当前节点相邻关系”
- 自动布局后保持当前选中对象不丢失

## 4. DevTools
### 页面目标
- 保持轻量 IDE 结构
- 强化对象上下文、结果扫描效率和资源切换连续性
- 不做“后台内嵌代码页”的廉价视觉

### 桌面布局
```text
+----------------------------------------------------------------------------------+
| SemanticPageHeader                                                               |
| 开发工具 | 当前对象 | 对象类型 | 最近编译结果                                     |
+----------------------------------------------------------------------------------+
| SemanticWorkbenchContextBar                                                      |
| 当前对象路径 | 最近执行时间 | Schema 状态 | [编译] [同步]                         |
+----------------------------------------------------------------------------------+
| Grid: 260px / minmax(0,1fr)                                                      |
| +----------------------+ +----------------------------------------------------+ |
| | ResourceTree         | | Workspace                                           | |
| | 搜索                 | | Tabs: 定义文件 / 编译调试 / Schema 同步            | |
| | Catalog / Domain     | |                                                    | |
| | Cube / View          | | 定义文件: Monaco/YAML                              | |
| | 最近访问             | | 编译调试: 结果摘要 + 日志列表 + 对象定位           | |
| |                      | | Schema 同步: 漂移状态 + 建议动作                   | |
| +----------------------+ +----------------------------------------------------+ |
+----------------------------------------------------------------------------------+
```

### Tabs 细化
- `定义文件`
  - 对 Cube / View 支持在线编辑
  - 对 Domain / Catalog 显示“当前不支持在线编辑”的产品化说明
- `编译调试`
  - 顶部结果摘要
  - 中部日志列表
  - 底部或侧边问题定位
- `Schema 同步`
  - 同步状态列表
  - 最近检查时间
  - 建议动作

### 视觉要求
- 编辑器和结果区统一采用 `Inset Surface`
- 资源树行高紧凑，但保持良好焦点态
- 顶部上下文条承担 IDE 语境，不再靠二次标题重复说明
- 结果状态优先可扫描，不用彩色大卡片

### 组件拆分
- `SemanticResourceTree.tsx`
- `SemanticWorkspaceHeader.tsx`
- `SemanticEditorEmptyState.tsx`
- `SemanticCompileSummary.tsx`
- `SemanticSchemaSyncPanel.tsx`

### 关键交互
- 切换对象时保留当前 tab
- 编译失败时自动切到 `编译调试`
- 无法在线编辑的对象，提供明确跳转建议
- 资源树支持最近访问和当前对象高亮

---

## 5. 实现边界
- 不改全局导航、顶部框架和路由
- 不引入新的重量级 UI 框架
- 不为线框说明新增后端能力假设
- 所有新增布局优先复用现有 `SemanticPageHeader`、`SemanticSurface`、`SemanticStatusBanner`、`SemanticInspectorPanel`

## 6. 统一前端实现约束
- URL 状态继续使用 `useUrlState`
- 页面查询继续基于 `TanStack Query`
- 画布继续使用 `React Flow + ELK`
- 编辑器继续使用 `Monaco`
- 响应式降级时不隐藏核心动作，只改变布局承载方式
