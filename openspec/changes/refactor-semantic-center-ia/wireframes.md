# 语义中心页面线框说明

本文件是 `refactor-semantic-center-ia` 的页面级可执行输入，直接服务于设计评审、前端开发拆分与验收。

统一原则：
- 页头只写功能，不写流程
- 上下文条只写范围、状态、数量
- 首屏只保留一个主任务区
- Inspector 只在当前任务需要时显示
- `KISS`：不新增一级导航，不重做主路由
- `YAGNI`：`View / Recipe` 作为次级资源挂载，不单独升格

## 1. Cube 管理页线框

页面类型：`Inventory`

页面目标：
- 帮用户快速判断当前 `Cube` 资产规模、状态分布和结构规模
- 让用户直接进入需要维护的 `Cube`

首屏必须回答：
- 现在一共有多少 `Cube`
- 它们当前处于什么状态
- 哪些 `Cube` 的维度 / 指标结构值得先看

桌面线框：

```text
┌──────────────────────────────────────────────────────────────────────┐
│ PageHeader                                                          │
│ 标题：Cube 管理                                                     │
│ 说明：查看 Cube 定义、状态和结构摘要                               │
│ 右侧：新建 Cube                                                     │
├──────────────────────────────────────────────────────────────────────┤
│ ContextBar                                                          │
│ Cube 总数 | 已发布 | 草稿 | 校验失败                               │
├──────────────────────────────────────────────────────────────────────┤
│ Toolbar                                                             │
│ [搜索____________________] [Cube 类型] [所属领域] [状态] [排序] [清空] │
│ [全部] [待处理] [未绑定] [未纳域] [最近更新]                        │
├───────────────────────────────────────────────────────┬──────────────┤
│ Object List                                           │ Inspector    │
│                                                       │              │
│ Cube 名称                                             │ 默认隐藏     │
│ code                                                  │              │
│ 一句说明                                              │ 选中后显示： │
│ 维度数 / 指标数 / View 数 / 所属领域                  │ 字段         │
│ 状态                         更新时间                 │ 属性         │
│ 编辑图标                                             │ 领域         │
│                                                       │ 当前状态     │
└───────────────────────────────────────────────────────┴──────────────┘
```

字段优先级：
1. `Cube 名称`
2. `状态`
3. `维度数 / 指标数`
4. `所属领域`
5. `更新时间`

明确弱化：
- 负责人
- 版本
- 来源长文本
- 预览默认常驻

交互规则：
- 选中行前，不显示右侧 Inspector
- 搜索和筛选保持单层
- 行内主动作统一为图标，不重复文字按钮
- 长文本自动省略，点击进入详情或编辑再看全量信息

移动端降级：
- Inspector 改底部抽屉
- 快筛 chips 横向滚动
- 主列表只保留：名称、状态、结构规模、编辑

## 2. 领域目录页线框

页面类型：`Inventory`

页面目标：
- 提供“目录 -> 领域 -> Cube -> Join” 的资源组织入口
- 让用户先看到领域下有哪些 `Cube`，再看到它们的关系状态

首屏必须回答：
- 当前目录下有哪些领域
- 当前领域下有哪些 `Cube`
- 当前领域里的 `Join` 关系是否完整

桌面线框：

```text
┌────────────────────────────────────────────────────────────────────────────┐
│ PageHeader                                                                │
│ 标题：领域目录                                                            │
│ 说明：查看目录结构、领域下的 Cube 和关系组织状态                         │
│ 右侧：进入领域建模 / 新建目录                                             │
├────────────────────────────────────────────────────────────────────────────┤
│ ContextBar                                                                │
│ 当前目录 | 领域数 | Cube 数 | Join 数 | 草稿领域数                        │
├───────────────────────┬────────────────────────────────────┬───────────────┤
│ Catalog Rail          │ Domain / Cube Main                │ Join Summary  │
│                       │                                    │               │
│ 目录名称              │ 当前目录下的领域列表              │ 当前领域摘要  │
│ 领域数量              │ - 领域名称                        │ Join 数       │
│ Cube 数               │ - 状态                            │ 关联 Cube 数  │
│ Join 数               │ - Cube 数                         │ 关键关系提示  │
│                       │                                    │ 打开画布      │
│                       │ 选中领域后显示：                  │               │
│                       │ 该领域下的 Cube 列表              │               │
└───────────────────────┴────────────────────────────────────┴───────────────┘
```

字段优先级：
1. 当前目录
2. 当前领域
3. 领域下的 `Cube`
4. `Join` 关系摘要

明确弱化：
- 目录治理口径过强的报表指标
- 发布率、空目录提醒等管理视角信息

交互规则：
- 左侧 rail 负责切目录
- 中间主区默认先看领域，再看领域下 `Cube`
- 右侧摘要默认显示 `Join 数`
- 右侧只做当前目录 / 当前领域的轻摘要，不做重详情页

移动端降级：
- 目录 rail 改顶部下拉
- Join 摘要并入中间列表顶部

## 3. 领域画布页线框

页面类型：`Canvas`

页面目标：
- 让用户优先看清 `Cube 名称` 和 `Join` 关系
- 在同一页完成关系检查、Join 编辑和发布前判断

首屏必须回答：
- 当前领域里有哪些 `Cube`
- 它们之间怎么连
- 哪些 `Join` 有问题

桌面线框：

```text
┌────────────────────────────────────────────────────────────────────────────┐
│ PageHeader                                                                │
│ 标题：当前领域名                                                          │
│ 说明：维护当前领域内的 Cube 关系、Join 和发布状态                         │
│ 右侧：自动布局 / 发布领域                                                 │
├────────────────────────────────────────────────────────────────────────────┤
│ ContextBar                                                                │
│ 当前领域 | 当前状态 | Cube 数 | Join 数 | 待处理问题数 | 未保存状态       │
├───────────────────────┬────────────────────────────────────┬───────────────┤
│ Cube Library          │ Canvas                             │ Inspector     │
│                       │                                    │               │
│ [搜索] [过滤]         │ 以名称优先的节点                   │ 默认：领域摘要│
│                       │                                    │               │
│ Cube 标题             │ [Cube 名称]                        │ 选节点：      │
│ 编码                  │ 主要显示 Join 连线                 │ Cube 摘要     │
│ 状态                  │                                    │               │
│ 维度 / 指标           │ 缺失 / 冲突 / 正常 三态高亮        │ 选边：        │
│                       │                                    │ Join 设置     │
└───────────────────────┴────────────────────────────────────┴───────────────┘
```

视觉重点：
1. `Cube 名称` 必须是节点内最强信息
2. `Join` 连线必须比节点内次信息更醒目
3. `Join` 高亮必须区分 `缺失 / 冲突 / 正常`
4. 节点内继续显示 `维度数 / 指标数`

Inspector 字段：
- 领域摘要态：
  - 名称
  - 状态
  - Cube 数
  - Join 数
  - 发布检查
- Cube 摘要态：
  - 名称
  - 编码
  - 状态
  - 维度数
  - 指标数
- Join 编辑态：
  - 源 Cube
  - 目标 Cube
  - 源字段
  - 目标字段
  - Join Type
  - Cardinality
  - 说明
  - 保存 / 删除

明确弱化：
- 节点内大段属性说明
- 过多维度/指标细项
- 画布外说明区

移动端降级：
- 画布保持主区
- 左资源库改抽屉
- Inspector 改右下抽屉

## 4. 开发工具页线框

页面类型：`Developer Workbench`

页面目标：
- 以 `YAML` 定义文件为主入口
- 把 `Recipe` 明确成 `Few-shot / DSL 教学样本`

首屏必须回答：
- 当前打开的是哪个对象
- 当前在看哪个 tab
- 当前对象能不能编辑

桌面线框：

```text
┌────────────────────────────────────────────────────────────────────────────┐
│ PageHeader                                                                │
│ 标题：开发工具                                                            │
│ 说明：查看定义文件、编译结果和 Schema 同步状态                             │
├────────────────────────────────────────────────────────────────────────────┤
│ ContextBar                                                                │
│ 当前对象 | 对象类型 | 当前标签页 | Schema 状态 | 资源规模                  │
├───────────────────────┬────────────────────────────────────────────────────┤
│ Resource Tree         │ Workspace                                          │
│                       │                                                    │
│ Cube                  │ Workspace Header                                   │
│ View                  │ 当前对象 / 当前状态 / 主动作                       │
│ Recipe                │                                                    │
│ Domain                │ Tabs                                               │
│ Catalog               │ [定义文件] [编译调试] [Schema 同步]                │
│                       │                                                    │
│                       │ Main                                               │
│                       │ 默认进入 YAML 编辑器                               │
│                       │                                                    │
│                       │ Recipe 时显示：                                    │
│                       │ 名称 / 标签 / 相关 Cube / 示例数 / YAML            │
└───────────────────────┴────────────────────────────────────────────────────┘
```

资源树优先级：
1. Cube
2. View
3. Recipe
4. Domain
5. Catalog

定义文件 tab 重点：
- 文件路径
- 未保存状态
- 校验
- 保存
- YAML 编辑器

`Recipe` 定位：
- 查询配方
- Few-shot 示例
- Agent 生成 DSL 的教学样本

因此 `Recipe` 在工作台内必须展示：
- 名称
- 标签
- 相关 Cube
- 示例数
- YAML
- 查看关联 Cube

明确弱化：
- 编译调试默认存在感
- 平均分配三个 tab 的权重

移动端降级：
- 资源树改左抽屉
- YAML 仍是默认 tab

## 5. 后续评审问题

本稿进入评审时，建议重点确认：
- 总览页的“最近操作记录”是否要区分对象类型
- Cube 管理页的 `Cube 类型` 是否固定为高优先级筛选
- 领域目录页右侧摘要中，除 `Join 数` 外是否还需要固定显示 `关联 Cube 数`
- 领域画布里 `Join` 三态高亮的颜色与图例是否需要单独规范
- 开发工具里 `Recipe` 的“查看关联 Cube”是内联展示还是跳到对象详情
